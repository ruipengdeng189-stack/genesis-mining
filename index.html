搭建方案

为什么不买数据库也能先跑起来

大厅先行 根目录只负责导航。每个游戏都单独放在 games/ 下。
本地存档 每个游戏先用 localStorage 存档。大厅在同域名下可以读取这些数据。
免费发布 你仍然使用 GitHub -> Vercel 自动发布流程。
后续再扩展 等你真的需要跨设备同步时，再补最轻量的数据层。
UI 素材清单

你后续可以准备的素材

大厅 Logo 建议规格：512x512 透明 PNG 或 SVG。
10 张游戏封面 建议规格：800x1000，最好保持统一风格。
大厅背景 建议主题：赛博、能源核心、科技机房、霓虹街机风格。
图标包 主页入口、排行、任务、商店、通行证、活动都需要统一图标体系。
音效包 点击、领奖、合成、确认、菜单、BGM 建议统一为科幻电子风。
特效素材 后续可以补徽章边框、发光纹理、奖杯素材、粒子蒙版等效果。
方案说明

当前为静态优先架构

这个大厅暂时不依赖付费数据库。它的作用是负责导航、读取各个游戏的本地存档，让你可以先把 10 个游戏入口搭起来，再一个一个填充真正的游戏内容。

1. index.html 做大厅首页
2. games/mining/ 放挖矿游戏
3. 其他游戏先用占位页
4. 共用成长先基于 localStorage
